# FPS 장기 핵심 시스템 구현 상태

## Steam 세션

`USessionSubsystem`이 다음 기능을 제공한다.

- `CreateRoom`: Steam 로비를 생성하고 `MainMap?listen`으로 이동
- `FindRooms`: `FPSJANGGI_GAME=FPSJANGGI_V1` 방 검색
- `JoinRoom`: 검색 결과의 접속 주소로 `ClientTravel`
- `QuickMatch`: 참가 가능한 방에 입장하고 없으면 새 방 생성
- `LeaveRoom`: 현재 세션 종료

개발용 Steam App ID는 480이다. 실제 전송 검사는 서로 다른 Steam 계정과 PC에서 수행해야 한다.

## 서버 권한 장기판

`AAuthoritativeJanggiBoard`가 유일한 규칙 판정자다.

- 서버가 `Start_Figure[90]`을 읽어 Blue/Red 기물 Blueprint 32개를 생성한다.
- 클라이언트는 좌표만 RPC로 요청하고 이동·충돌·제거는 서버가 결정한다.
- 졸, 차, 포, 마, 상, 사, 궁의 행마법과 길막을 C++에서 검증한다.
- 턴, 클릭 기록, 제한 시간, 보드 정지, 전투 상태를 복제한다.
- 기본 턴 제한은 60초이며 만료 시 선택을 취소하고 상대 턴으로 넘긴다.
- 같은 기물의 기본 연속 이동 제한은 2회다.

`origin/master`의 원본 `JanggiBoard1` 값을 사용한다.

- 좌표: `X=(index / 9) * 8.0`, `Y=(index % 9) * 8.5`
- 기물 Actor 스케일: `1.8`

## 진형 선택

기존 로컬 전용 `InterFace_MS` 대신 C++ 진형 선택 UI를 사용한다. 양쪽 선택이 서버에 도착할 때까지 보드 입력을 정지하며, 선택이 완료되면 32개 기물을 생성하고 Blue 턴을 시작한다.

장기에서는 시작 전에 각 플레이어가 뒷줄 좌우의 서로 붙어 있는 마와 상 위치를 바꿀 수 있다. 따라서 `마-상-상-마`, `마-상-마-상`, `상-마-상-마`, `상-마-마-상` 네 배치는 정상 규칙이다. 이 선택은 마/상의 순서만 바꾸며 청·한 진영의 보드 끝이나 카메라 방향은 바꾸지 않는다.

## 이동 가능 칸 표시

기물을 선택하면 서버가 현재 보드 점유 상태와 실제 행마법으로 90칸을 검사한다. 합법적인 목적지 인덱스만 해당 플레이어에게 Client RPC로 전송한다.

- 목적지는 장기판 위의 밝은 초록색 구형 마커로 표시한다.
- 화면 좌표 UMG 투영을 사용하지 않는다. 각 합법 칸의 실제 장기판 로컬 좌표 위에 엔진 기본 구체를 작게 배치하고 녹색 동적 재질과 명시적 게임 가시성을 적용해 카메라 각도와 화면 해상도에 관계없이 교차점 위에서 보이게 한다.
- 마커는 로컬 클라이언트에만 생성하고 충돌을 비활성화하므로 보드 클릭이나 네트워크 복제에 영향을 주지 않는다.
- 화면 재투영 타이머가 없으며 기물을 선택할 때만 생성하고 선택 해제·이동·턴 전환 시 제거한다.
- 같은 편의 다른 기물을 선택하면 목록을 즉시 갱신한다.
- 이동, 턴 만료, 턴 전환, 아레나 진입 시 모두 숨긴다.
- 이동·턴 전환 시 화면 마커를 즉시 제거한다.
- 상대 기물, 빈 칸 또는 불가능한 목적지를 누르면 상단에 거절 이유를 1.5초 동안 표시한다.

## 턴 표시 UI

각 로컬 플레이어 화면 상단에 다음 정보를 표시한다.

- `내 차례` 또는 `상대 차례`
- 현재 팀: 청 또는 한
- 남은 턴 시간
- 진형 선택/상대 대기 상태
- 기물 충돌 전투 진행 상태

Blue 턴은 파란 배경, Red 턴은 붉은 배경으로 표시한다. UI는 `HitTestInvisible`이므로 보드 클릭을 방해하지 않는다.

## 카메라

MainMap에 원래 배치된 두 카메라를 유지한다.

- Blue: `CameraActor_1`
- Red: `CameraActor_2`
- 배치 카메라의 위치·시야각·바라보는 방향을 기준으로 팀별 로컬 카메라를 만들고, 자기 진영이 항상 화면 아래쪽이 되도록 화면 상하축만 자동 보정한다. 진형 선택 결과와는 무관하다.
- 아레나 진입 직전 각 로컬 플레이어의 실제 ViewTarget을 저장하고, 종료 후 팀 이름을 다시 추측하지 않고 저장한 카메라로 복귀한다.

## 충돌과 아레나 전환

상대 기물이 있는 합법 목적지를 선택하면 보드 입력을 정지하고 `ArenaTransition`으로 전환한다. 전투 시스템은 `ResolveArenaBattle(WinnerTeam)`을 호출해 결과를 서버에 전달한다.

현재 전투 시스템 연동 전 임시 단계에서는 충돌한 기물 종류에 대응하는 스켈레탈 메시 캐릭터 두 명만 생성한다.

- 청/한 임시 캐릭터와 양쪽 카메라 위치를 배치된 `JanggiBoard1` 인스턴스의 Details 패널에서 직접 지정할 수 있다.
- `Arena Blue Spawn Offset`, `Arena Red Spawn Offset`: 두 임시 캐릭터의 위치
- `Arena Blue Camera Offset`, `Arena Red Camera Offset`: 각 팀이 전투 중 사용할 카메라 위치
- `Arena Camera Focus Offset`: 양쪽 전투 카메라가 바라볼 위치
- 다섯 값은 장기판 Actor 기준 상대 좌표이며 `Make Edit Widget`이 적용되어 레벨 뷰포트의 3D 핸들로도 옮길 수 있다.
- 임시 캐릭터의 메시와 위치는 서버에서 생성·복제
- Development 빌드에서만 하단에 `청 승리로 전투 종료`, `한 승리로 전투 종료` 버튼 표시
- 디버그 승리 요청도 서버가 검증하고 `ResolveArenaBattle`을 호출

- 공격자 승리: 수비 기물 제거 후 공격 기물 이동
- 수비자 승리: 공격 기물 제거
- 처리 후 상대 턴 시작 및 기존 보드 카메라 복귀
- 패배한 기물이 궁/왕이면 즉시 게임을 종료하고 승리 팀을 양쪽 화면에 표시하며 보드 입력과 턴 타이머를 정지

### 전투 위치를 에디터에서 지정하는 방법

1. PIE를 종료한다.
2. World Outliner에서 레벨에 배치된 `JanggiBoard1` Actor를 선택한다.
3. Details에서 `FPS Janggi > Arena > Placement`를 연다.
4. 청/한 스폰, 청/한 카메라, 카메라 초점의 다섯 위치를 숫자로 입력하거나 뷰포트의 위치 핸들을 드래그한다.
5. 플레이해 양쪽 화면을 확인한 뒤 카메라가 캐릭터를 바라보도록 스폰과 초점 위치를 미세 조정한다.

이 설정은 레벨에 저장되므로 팀원이 같은 `MainMap`을 받으면 같은 전투 위치를 사용한다.

## 성능 최적화

- 기물 선택마다 월드의 모든 Actor를 검색하던 경로를 제거했다.
- 서버는 생성 시점부터 90칸 기물 캐시를 유지하고 이동·제거 때 해당 칸만 갱신한다.
- 복제 복구가 필요할 때도 `BP_Piece_Base` 하위 기물만 검색한다.
- 기물 스케일 동기화는 32개가 확인되는 즉시 반복 타이머를 종료한다.
- PlayerController가 장기판 Actor를 한 번 찾은 뒤 캐시한다.
- 턴 UI는 0.25초마다 확인하지만 텍스트나 색이 바뀔 때만 Slate 속성을 갱신한다.
- 하드웨어 레이트레이싱 지오메트리와 효과는 런타임에서 비활성화한다.
- 동적 GI와 반사는 SSGI/SSR을 사용하고 Virtual Shadow Map 대신 일반 Shadow Map을 사용한다.
- 텍스처 스트리밍 풀은 1500MB로 설정하고 실제 VRAM 한도를 준수한다.

## 수동 테스트 절차

1. 에디터를 완전히 종료했다가 다시 실행한다. 렌더링 설정 변경은 재시작해야 적용된다.
2. 플레이 옵션에서 `Number of Players=2`, `Net Mode=Play As Listen Server`를 선택한다.
3. 양쪽에서 서로 다른 진형을 선택한다.
4. Blue 창 상단에 `내 차례 · 청`, Red 창에 `상대 차례 · 청`이 표시되는지 확인한다.
5. Blue 기물을 클릭해 합법 목적지에 초록색 원형 마커가 나타나는지 확인한다.
6. 마커가 없는 칸을 클릭했을 때 기물이 움직이지 않는지 확인한다.
7. 마커가 있는 빈칸을 클릭해 기물이 이동하고 양쪽 화면에서 위치가 같은지 확인한다.
8. 이동 후 Blue 창은 `상대 차례 · 한`, Red 창은 `내 차례 · 한`으로 바뀌는지 확인한다.
9. 상대 기물이 있는 합법 마커를 선택해 보드가 정지하고 아레나로 전환되는지 확인한다.
10. 청/한 캐릭터가 전장 양 끝에 생성되고 두 플레이어가 각각 자기 캐릭터 뒤에서 중앙을 보는지 확인한다.
11. 하단의 `청 승리로 전투 종료` 또는 `한 승리로 전투 종료` 버튼을 누른다.
12. 승자에 따라 보드 기물이 이동/제거되고 임시 캐릭터와 디버그 UI가 사라지는지 확인한다.
13. 양쪽 모두 원래 보드 카메라로 돌아오고 다음 턴이 시작되는지 확인한다.
14. 궁/왕이 패배하는 충돌에서는 다음 턴으로 넘어가지 않고 양쪽 상단에 게임 종료와 승리 팀이 표시되는지 확인한다.
15. 게임 종료 뒤 기물을 클릭해도 선택·이동이 되지 않는지 확인한다.
16. 화면 좌측 상단에 레이트레이싱 메모리 초과와 VSM 큐 오버플로 경고가 다시 나타나지 않는지 확인한다.
17. 실제 Steam 검사는 패키징된 빌드와 서로 다른 Steam 계정 두 개로 진행한다.
