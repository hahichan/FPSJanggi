# YJH 다음 에이전트용 간단 프롬프트

아래 규칙을 지키며 FPSJanggi 전투/스킬 시스템 작업을 수행하라.

1. 시작 문서
- yjh_agent_first_task_template.md 체크리스트를 작업 단위마다 채운다.

2. 필수 원칙
- 하드코딩을 사실상 0에 가깝게 유지한다. 캐릭터별 분기/상수는 DataAsset 또는 설정으로 이동한다.
- 파이프라인은 입력 -> 검증 -> 실행 -> 반영 -> 이벤트 단계로 명확히 분리한다.
- 코드 가독성과 유지보수성을 우선한다. 함수 책임을 작게 유지하고 실패코드는 표준 FailCode만 사용한다.
- 서버 권한을 고정한다. 최종 판정/종료/쿨타임 확정은 서버에서만 수행한다.
- SkillData는 읽기 전용, 동적값은 RuntimeContext에서만 사용한다.

3. 목표
- 타 개발자가 코드 수정 최소화로 신규 캐릭터를 쉽게 만들 수 있게 한다.
- 신규 캐릭터 추가는 BP/DataAsset 중심으로 가능해야 한다.
- 작업 후 yjh_chat_4.md에 작업 번호로 변경 사항을 기록한다.

4. 출력 형식
- 변경 파일 목록
- 파이프라인 변경 요약
- 하드코딩 제거/회피 근거
- 테스트 결과와 잔여 리스크
