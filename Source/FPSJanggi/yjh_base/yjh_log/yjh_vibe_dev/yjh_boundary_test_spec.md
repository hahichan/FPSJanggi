# YJH 책임 경계 테스트 명세서 (Step 2)

## 0. 목적
- 게임모드와 베이스 캐릭터의 책임 경계 위반을 조기에 검출한다.
- 구현 초기부터 경계 테스트를 고정해 구조 붕괴를 방지한다.

## 1. 경계 원칙
- 게임모드 책임: 세션 시작/종료, 승패 확정(FinalizeOnce), 보드 반영
- 캐릭터 책임: 이동/입력/피해 전달, 사망 이벤트 보고
- 금지: 캐릭터가 승패 확정 로직 수행, 게임모드가 캐릭터 내부 수치 직접 조작

## 2. 테스트 케이스
### TC-BND-01 세션 시작 경계
- 목적: BeginCombatSession 호출이 게임모드 오케스트레이션 경로로만 시작되는지 확인
- 기대 결과: 캐릭터는 준비/수신만 수행하고 세션 결정권 없음

### TC-BND-02 피해 처리 경계
- 목적: 피해 계산/적용은 캐릭터 컴포넌트에서 처리되고 게임모드는 결과 수신만 하는지 확인
- 기대 결과: 게임모드가 CurrentHP를 직접 수정하지 않음

### TC-BND-03 종료 확정 경계
- 목적: FinalizeCombatSession이 서버 단일 지점에서 1회만 실행되는지 확인
- 기대 결과: 중복 ReportCombatantDead 수신 시 Finalize 재실행 없음

### TC-BND-04 이탈 처리 경계
- 목적: 이탈/타임아웃은 게임모드 AbortCombatSession으로 일원화되는지 확인
- 기대 결과: 캐릭터 단에서 세션 종료 확정 수행 금지

### TC-BND-05 확장성 경계
- 목적: 파생 캐릭터 추가 시 Base/게임모드 수정 최소화 확인
- 기대 결과: 신규 파생 캐릭터 추가에 필수 C++ 수정 파일 3개 이하

## 3. 실패 판정 규칙
- 경계 위반 발견 시 해당 항목 즉시 Block 처리
- Block 항목 해소 전 다음 단계(3~6단계) 진행 금지

## 4. 실행 체크 항목
1. 서버/클라 로그에서 함수 호출 주체 확인
2. Finalize 관련 로그 1회성 확인
3. 파생 캐릭터 샘플 생성 시 수정 파일 수 집계

## 5. 추적 로그 키
- YJH_BND_SESSION_START
- YJH_BND_DAMAGE_FLOW
- YJH_BND_FINALIZE_ONCE
- YJH_BND_ABORT_FLOW
- YJH_BND_EXTENSIBILITY
