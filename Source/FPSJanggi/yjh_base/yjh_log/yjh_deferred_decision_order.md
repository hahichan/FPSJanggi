# YJH 보류 항목 결정 순서표 (Step 6, Draft)

## 0. 목적
- 구현 꼬임을 줄이기 위해 보류 항목의 결정 순서와 진입 조건을 고정한다.
- 본 문서는 계획 단계 Draft이며 실제 코드 작성 전까지 변경 가능하다.

## 1. 보류 항목
- 데미지 공식
- 히트 판정 방식
- 입력 버퍼/콤보 규칙
- 애니메이션 노티파이 연동

## 2. 결정 순서(고정)
1. 데미지 공식
2. 히트 판정 방식
3. 애니메이션 노티파이 연동
4. 입력 버퍼/콤보 규칙

## 3. 순서 근거
- 데미지 공식이 수치 기준을 만든다.
- 히트 판정 방식이 데미지 적용 조건을 만든다.
- 노티파이 연동은 판정/적용 타이밍을 시각 이벤트에 연결한다.
- 입력 버퍼/콤보는 위 3요소가 안정화된 뒤 확장해야 리스크가 낮다.

## 4. 항목별 진입 조건
### 1) 데미지 공식
- 최소 수직 슬라이스 3회 성공
- HP 동기화 이슈 없음

### 2) 히트 판정 방식
- 데미지 공식 v0.1 확정
- FinalizeOnce 중복 이슈 없음

### 3) 애니메이션 노티파이 연동
- 히트 판정 이벤트 타이밍 로그 확보
- 클라 지연 환경에서 오차 허용 범위 정의

### 4) 입력 버퍼/콤보 규칙
- 노티파이 기반 공격 타이밍 안정
- 확장성 가이드에 콤보 확장 지점 문서화 완료

## 5. 산출물 규칙
- 각 항목 결정 시 문서 3종 동기화:
  1. yjh_plan_base_character.md
  2. yjh_plan_integration_contract.md
  3. yjh_dev_execution_checklist.md

## 6. 실패/되돌림 규칙
- 항목 결정 후 회귀 발생 시 직전 항목 상태로 롤백
- 같은 항목 재시도는 원인/가설/검증 절차를 로그에 남긴 뒤 진행
